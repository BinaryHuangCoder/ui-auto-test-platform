## 🔑 首次使用 - 配置 SSH 免密登录

### 步骤 1: 生成 SSH 密钥 (如果还没有)

```bash
ssh-keygen -t ed25519
# 一路回车即可
```

### 步骤 2: 复制公钥到服务器

```bash
# 使用 sshpass (需要密码)
sshpass -p "你的密码" ssh -o StrictHostKeyChecking=no root@10.254.239.205 \
  "mkdir -p ~/.ssh && cat >> ~/.ssh/authorized_keys" < ~/.ssh/id_ed25519.pub

# 或使用 ssh-copy-id (如果可用)
ssh-copy-id -o StrictHostKeyChecking=no root@10.254.239.205
```

### 步骤 3: 验证免密登录

```bash
ssh root@10.254.239.205 "echo 'SSH 免密登录成功'"
```

## 🚀 快速部署

## ✨ 部署流程

`server-deploy.sh` 脚本会自动执行以下步骤：

1. ✅ 停止服务器上的旧服务

   - 创建数据库 `ui_auto_test`
   - 导入初始化 SQL
   - 配置 root 远程访问
3. ✅ 编译后端 (Maven + JDK 1.8)
4. ✅ 清理并重新安装前端依赖
5. ✅ 修复前端兼容性问题（vite.config.js、user.js）
6. ✅ 编译前端 (Vite + Node.js 16.18.1)
7. ✅ 配置 Nginx 反向代理（端口 8080）
8. ✅ 创建服务管理脚本
9. ✅ 启动后端服务（端口 8888）
10. ✅ 启动前端服务（Nginx 端口 8080）
11. ✅ 验证部署状态

## 🐛 故障排查

### SSH 免密失败

```bash
# 检查公钥
ssh root@10.254.239.205 "cat ~/.ssh/authorized_keys"

# 重新配置
sshpass -p "你的密码" ssh root@10.254.239.205 \
  "mkdir -p ~/.ssh && chmod 700 ~/.ssh && cat >> ~/.ssh/authorized_keys" < ~/.ssh/id_ed25519.pub
ssh root@10.254.239.205 "chmod 600 ~/.ssh/authorized_keys"
```

### 后端编译失败

```bash
ssh root@10.254.239.205 "
  cd /opt/ui-auto-test-platform/backend
  export JAVA_HOME=/usr/java/jdk1.8.0_172-amd64
  /opt/maven/apache-maven-3.8.6/bin/mvn clean package -DskipTests
"
```

### 登录 405 错误

确保 Nginx 正确配置了 API 代理：

```bash
ssh root@10.254.239.205 "cat /usr/local/nginx/conf/nginx.conf"
```

检查是否有：
```nginx
location /api/ {
    proxy_pass http://127.0.0.1:8888/api/;
    ...
}
```

### 前端 6666 端口无法访问

Chrome 浏览器将 6666 端口视为不安全端口（ERR_UNSAFE_PORT）。解决方案：
1. 使用端口 8080 或 3000
2. 或使用 Firefox/Edge 浏览器访问
